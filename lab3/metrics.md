# Метрики времени работы приложения

Hadoop приложение представляет из себя кластер, где каждая нода, в том числе datanode, представлена в одном экземпляре (ограничения модели с использованием docker). Поэтому эксперименты будут нацелены на изменение количества строк при логической группировке данных на стадии map.

Параметр при измерении времени работы:

- Количество строк, которые передаются mapper

Результаты

| Кол-во строк | Время работы |
| --- | --- |
| 10'000 | 1m 42s |
| 50'000 | 1m 42s |
| 100'000 | 1m 42s |
| 500'000 | 1m 42s |
| 1'000'000 | 1m 42s |

# Результат

Логическое разделение данных для map без параллелизма не ускоряет работу системы
